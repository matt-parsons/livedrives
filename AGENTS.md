# Agent Guidelines

## Scope
These guidelines apply to the entire repository.

## Code Style
- Follow the existing formatting conventions in each file; do not introduce new styles unless necessary.
- Prefer descriptive variable and function names that communicate intent clearly.

## Testing
- Run relevant test suites or lint commands after making code changes.
- Include the commands executed and their results in the final summary.

## Documentation
- Update or create documentation when behavior changes.

## Pull Requests
- Provide concise, actionable summaries describing the changes and tests executed.
